# FitMan 版本更新信息

## v0.4.0 — 大卡片聚焦流重构 (2026-07-01)

### 新增

**训练页完全重构：大卡片聚焦流**
- 一次只显示一组，点[完成]→倒计时→自动推进下一组
- 重量智能沿用：同动作后续组自动填入上一组重量，切换动作清空
- 顶部分段进度条：绿色=完成、脉动=当前、虚线框=跳过、灰=待做
- 进度条可点击已完成的动作，弹出编辑面板修改任意组数据

**跳过与补做**
- 「跳过这组」按钮：标记 skipped 直接推进，不走倒计时
- 「跳过这个动作」按钮：确认弹窗后跳过该动作所有剩余组
- 训练结束页：展示已完成/跳过统计，列出跳过的动作
- 补做模式：结束页进入，只遍历 skipped 的组，逐一补做

**自定义组间休息**
- 模板编辑页每个动作新增「休息(s)」字段，默认 60s
- 训练时自动使用模板预设休息时长
- 倒计时页面提供 ±15s 快速微调

**断点保护**
- 每组完成时实时写入 localStorage，应用被杀后可恢复
- 启动时检测未完成训练，弹窗询问继续/放弃
- 恢复时智能判断推进位置（已完成的组自动略过）

### 变更
- 数据模型：`done: true/false` → `status: "pending" | "done" | "skipped"`
- History 页兼容新旧数据格式
- 计时器 ±15s 微调按钮

### 设计文档
- ROADMAP.md 包含完整设计方案，含页面布局、交互流程、状态机、数据模型

## v0.3.1 — APK 黑屏修复 (2026-06-30)

### 修复
- **APK 黑屏**：`useMemo` 在 React 19 + Capacitor 8 WebView 中导致渲染崩溃，全部改用 `useEffect` + `useState`
- 数据安全：`JSON.parse` 和热力图数据处理加 `try-catch`
- 覆盖安装旧数据兼容：防御性检查 `Array.isArray`、null 守卫

### 教训
- 本项目 Capacitor APK 场景禁用 `useMemo` / `useCallback`，统一用 `useEffect` + `useState`

## v0.2.0 — 计时器改版 + PWA 完善 (2026-06-29)

### 新增
- 组间计时器新增 30 秒选项（现为 30/60/90/120 四档）
- 计时开始后全屏覆盖显示倒计时，结束后自动返回训练页
- PWA 完整支持：manifest.json、Service Worker 离线缓存、Apple 全屏模式

### 修复
- APK 白屏问题：构建路径改为相对路径，APK 与网页版分离构建
- Service Worker 路径自适应，APK 和网页通用

### 构建命令
- `npm run build` — APK 用（`base: ./`）
- `npm run build:web` — 网页用（`base: /fitman2/`）
- `npm run sync:android` — 构建 + 同步到安卓项目
- `npm run deploy` — 构建网页 + 部署到 GitHub Pages

## v0.1.0 — 初始版本 (2026-06-29)

### 已有功能
- 首页：日期显示、今日训练状态、快速开始模板/空白训练、最近训练记录
- 训练页：动作管理（增删改名）、逐组记录重量×次数、完成勾选、组间计时器（60/90/120s）、计时结束震动+闪烁、训练完成一键保存
- 历史页：训练记录按日期列表查看
- 模板页：训练模板创建/编辑/删除
- 底部导航栏四个 Tab 切换
- 数据存储在浏览器 localStorage

### 部署
- GitHub Pages: https://silver1182.github.io/fitman2/
- 安卓 APK: Capacitor 打包，debug 版本
- PWA: 支持添加到主屏幕全屏运行，Service Worker 离线缓存

### 技术栈
- React + Vite
- Capacitor (Android)
- PWA (manifest.json + Service Worker)
